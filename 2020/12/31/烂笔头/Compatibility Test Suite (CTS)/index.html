Compatibility Test Suite (CTS)

发表于 2020-12-31 更新于 2020-01-16 分类于 烂笔头 阅读次数：

官网资源：

http://source.android.com/compatibility/downloads.html

  1. 什么要过CTS ？

  2. Give your users the best possible experience with the applications they run.

When a device is compatible with Android, users can choose from among many 

high-quality applications. Applications that take full advantage of Android’s 

features are likely to perform best on compatible devices.

2.

Make it easy for developers to write top-quality applications for your device.

Developers want to streamline their applications for Android, and this is easiest for 

them when they are writing for a predictable platform. 

3.

Take advantage of the Android Market.

Compatible handsets can give users access to the Android Market.

  1. adb 环境

RK的SDK直接执行setpaths , 或参考Android官网

3.拷贝媒体文件

CTS Media 1.0

执行 copy_media.sh

4.设置及安装测试APK

Settings > Wi-Fi

Settings > Display > Sleep > never timeout

Settings > Security > Screen Lock  should be ‘None’

Settings > Developer options > USB debugging

Settings > Developer options > Stay Awake is checked

Settings > Developer options > Allow mock locations is checked

Settings > Location services > Google’s location service (network location needed)

Settings > Data & time > Select time zone (some timezone doesn’t work)

adb install android-cts/repository/testcases/CtsDelegatingAccessibilityService.apk

enable Settings > Accessibility > Delegating Accessibility Service

adb install android-cts/repository/testcases/CtsDeviceAdmin.apk

enable Settings > Security > Device Administrators > android.deviceadmin.cts.CtsDeviceAdmin* settings

Make sure the device is at the home screen at the start of CTS

5. ./android-cts/tools/cts-tradefed   CTS 命令

  执行  run cts –plan CTS 

  完成后生成报告 ：  $CTS_ROOT/repository/results/<start time>.zip  

run cts

    –plan <test_plan_name>

    – package/-p <test_package_name>  [–package/-p <test_package2>…]

    –class/-c <class_name> [–method/-m <test_method_name>

    –continue-session-id

    –shards  <number_of_shards>

    –serial/-s <deviceID>

List

    list packages

    list plans

    list invocations

    list commands

    list results  (l r)

    list devices

Add  （添加新计划）

add derivedplan –plan <plan_name>

–result/-r

[pass | fail | timeout | notExecuted]

[–session/-s <session_id>]

CTS-verifier

     Data Backup test

          –>settings->Backup my data 勾选

    进入Data Backup test -> Generate Test Data

  adb shell
                bmgr enable true

                bmgr transport android/com.android.internal.backup.LocalTransport

  卸载Cts-verifier,重新安装，数据一样表示通过。

需要注意的地方：

  1. system/etc/permissions

 相应权限必须对应，系统没有的功能，不能出现

 如 <feature name="android.hardware.sensor.accelerometer" />

在 hardware/rk29/sensor/st   sSensorList[] 对应定义

 <feature name="android.hardware.camera" />

<feature name="android.hardware.camera.front" />

     <feature name="android.hardware.location" />

<feature name="android.hardware.location.network" />

前置摄象头与网络定位如此声明

2. media_profiles.xml 与 kernel 中 定义摄像头的帧率须相同

  1. 系统某些必须的APK是不能删除的，如 Phone.apk Gsmcore.apk 等
